# avito-test-applicant

Решение тестового задания на стажировку Авито, осень 2025

> Сервис для автоматического назначения ревьюеров на Pull Request (PR) в рамках команды, с возможностью переназначения и получения списка PR для пользователей.

## **Запуск**

Сервис полностью поднимается через Docker Compose:

```bash
docker compose up
```

После этого сервис доступен на порту `8080`.

> По адресу http://localhost:8080/docs/swagger доступна интерактивная документация

## **Особенности реализации**

-   **UUID** - принял контракт, что сервер принимает ID как строки, которые преобразуются в UUID. В базе и коде используются реальные UUID. Такое решение принял для упрощения генерации идентификаторов и обеспечения единообразия ID. Также обновил `docs/openapi.yml`, актуализировал примеры с использованием ID.
-   Для генерации API-хендлеров и типов использовался oapi-codegen. Так я автоматически синхронизировал реализацию сервиса с OpenAPI-спецификацией.

## **Тестирование**

-   Интеграционные тесты покрывают repo- и service- логику. Для запуска:

```bash
make test
```

> Для интеграционных тестов используется testcontainers для поднятия PostgreSQL
